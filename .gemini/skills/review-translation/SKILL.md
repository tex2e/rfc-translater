# RFC翻訳内容レビュー手順

1. `rfcNXXX-trans.json` の翻訳ファイルについて、`ja` フィールドの日本語の内容が `text` の原文と比較して、正しく翻訳されているか確認し、必要があれば日本語を修正してください。
    - 基本はreplaceツールで置換してください。replaceツールが何回も失敗するときは代わりにPython3で `scripts/fix_rfc.py` を参考に修正してください。
    - 修正時はJSONの構造を壊さないように、ダブルクオーテーションはバックスラッシュでエスケープするなどの処理をしてください。
    - 文脈から一つの名詞句（例：application/geo+json-seq）と判断できるものについては、「/」や「+」の前後にある空白を除去してください。
2. `section_title` が true のときは、`ja` フィールドの日本語の内容が体言止めで見出しとして適切な内容かを確認し、必要があれば日本語を修正してください。
3. 翻訳内容の修正が完了したら、最後に全体を俯瞰して、技術者が読んだ時に1つのシステムとして動く仕様書になっているか机上で検証してください。
4. 最後に、`rfcNXXX-summary.json` が存在する場合のみ `summary` の内容が概要として正しいか検証し、必要に応じて修正してください。
    - このRFCが位置するネットワーク階層、歴史的背景、そしてコアとなる目的をWeb検索などを使って特定してから修正してください。
    - ただし要約は3行で200文字程度で収まるようにしてください。
