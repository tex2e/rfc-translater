
このレポジトリは、翻訳済みRFCをJSON形式で管理しています。

## プロジェクト構造

- `data/N000/rfcNXXX.json` : 段落区切りの文書
- `data/N000/rfcNXXX-trans.json` : 各文章の翻訳を付与した情報
- `data/N000/rfcNXXX-summary.json` : RFCの要約

### 要件定義

- 内容
    - 文章のみ翻訳し、図や表やサンプルコードはそのまま表示すること
    - 文章がページ区切りで分割されていても1つの段落として翻訳すること
    - インデントの深さも表示に反映させること
    - 箇条書き（o + * - など）の記号はそのまま表示すること（翻訳時は箇条書き記号より後の文字列だけを翻訳する）
    - 表題（1.2.～ など）は見出しとして文字を大きくすること
- `rfcNXXX-trans.json` のJSON構造
    - `title`: タイトルデータ（オブジェクト）
        - `text` : 英語の原文（文字列）。先頭は必ず「RFC XXXX - 」から始まる
        - `ja` : 翻訳後の日本語（文字列）。タイトルのため体言止め。先頭は必ず「RFC XXXX - 」から始まる
    - `contents` : 各段落の内容（配列）
        - `text` : 英語の原文（文字列）
        - `ja` : 翻訳後の日本語（文字列）。箇条書き以外のときは、ですます調で表現すること
        - `indent` : 表示するときのインデント数（整数）
        - `raw` : 図表やプログラムのフラグ（true: 翻訳せずにそのまま表示）
        - `section_title` : 章節項の見出しフラグ（true: 見出しタイトル）
- `rfcNXXX-summary.json` のJSON構造
    - `number`: RFCの番号（整数）
    - `model`: 要約を生成したAIのモデル（Geminiで出力したときは "gemini-3-flash"）
    - `created_at`: 要約生成日時（Geminiで出力したときは "2026-02-15T00:00:00.000000" 形式で今日の日時）。summaryを修正したときのみ更新すること
    - `"summary": [` : 要約内容（配列）... インデントは2
        - 要約文（文字列。最大3行程度。ですます調。強調の`**`やコードブロックの記号（バックスラッシュ）は使わないこと） ... インデントは4
    - `]` : インデントは2
