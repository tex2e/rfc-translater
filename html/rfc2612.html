Abstract

1. Introduction
1. はじめに

This document describes the CAST-256 encryption algorithm, a DES-like Substitution-Permutation Network (SPN) cryptosystem built upon the CAST-128 encryption algorithm [1] which appears to have good resistance to differential cryptanalysis, linear cryptanalysis, and related-key cryptanalysis. This cipher also possesses a number of other desirable cryptographic properties, including avalanche, Strict Avalanche Criterion (SAC), Bit Independence Criterion (BIC), no complementation property, and an absence of weak and semi-weak keys. It thus appears to be a good candidate for general-purpose use throughout the Internet community wherever a cryptographically-strong, freely-available encryption algorithm is required.

このドキュメントでは、CAST-256暗号化アルゴリズム、DES-128暗号化アルゴリズム[1]に基づいて構築されたDES-like置換透過ネットワーク（SPN）暗号システムについて説明します。暗号化。この暗号には、雪崩、厳密な雪崩基準（SAC）、ビット独立基準（BIC）、補完特性なし、弱いセミウィークキーの欠如など、他の多くの望ましい暗号化特性もあります。したがって、暗号化された自由に利用可能な暗号化アルゴリズムが必要な場合は、インターネットコミュニティ全体で汎用使用の良い候補であるように見えます。

CAST-256 has a block size of 128 bits and a variable key size (128, 160, 192, 224, or 256 bits).

CAST-256のブロックサイズは128ビットと可変キーサイズ（128、160、192、224、または256ビット）です。

The following notation from CAST-128 [1] is relevant to CAST-256.

CAST-128 uses a pair of subkeys per round: a 5-bit quantity Kri is used as a "rotation" key for round i and a 32-bit quantity Kmi is used as a "masking" key for round i.

CAST-128はラウンドごとにサブキーのペアを使用します。5ビット数量KRIは、ラウンドIの「回転」キーとして使用され、32ビット数量KMIはラウンドIの「マスキング」キーとして使用されます。

Three different round functions are used in CAST-128. The rounds are as follows (where D is the data input to the operation, Ia - Id are the most significant byte through least significant byte of I, respectively, Si is the ith s-box (see Section 2.1.1 for s-box contents), and O is the output of the operation). Note that "+" and "-" are addition and subtraction modulo 2**32, "^" is bitwise eXclusive-OR, and "<<<" is the circular left-shift operation.

CAST-128では、3つの異なるラウンド関数が使用されています。ラウンドは次のとおりです（Dは操作へのデータ入力です。IA-IDは、Iのそれぞれ最も重要なバイトを介して最も重要なバイトです。SiはITH S-Boxです（S-Boxについてはセクション2.1.1を参照してください内容）、およびoは操作の出力です）。"" and " - "は追加および減算モジュロ2 ** 32であることに注意してください。^"はBitWise Exclusive-orであり、「<<<」は円形の左シフト操作です。 

           Type 1:  I = ((Kmi + D) <<< Kri)
                    O = ((S1[Ia] ^ S2[Ib]) - S3[Ic]) + S4[Id]
        
           Type 2:  I = ((Kmi ^ D) <<< Kri)
                    O = ((S1[Ia] - S2[Ib]) + S3[Ic]) ^ S4[Id]
        
           Type 3:  I = ((Kmi - D) <<< Kri)
                    O = ((S1[Ia] + S2[Ib]) ^ S3[Ic]) - S4[Id]

2.2 CAST-256 Notation
2.2 CAST-256表記

The following notation is employed in the specification of CAST-256.

次の表記は、CAST-256の仕様に使用されています。

Let f1, f2, f3 be as defined for CAST-128.

f1、f2、f3をCAST-128に対して定義してください。 

       Let BETA = (ABCD) be a 128-bit block where A, B, C and D are each
       32 bits in length.
        
       Let "BETA <- Qi(BETA)" be short-hand notation for the following:
           C = C ^ f1(D, Kr0_(i), Km0_(i))
           B = B ^ f2(C, Kr1_(i), Km1_(i))
           A = A ^ f3(B, Kr2_(i), Km2_(i))
           D = D ^ f1(A, Kr3_(i), Km3_(i))
        
       Let "BETA <- QBARi(BETA)" be short-hand notation for the
       following:
           D = D ^ f1(A, Kr3_(i), Km3_(i))
           A = A ^ f3(B, Kr2_(i), Km2_(i))
           B = B ^ f2(C, Kr1_(i), Km1_(i))
           C = C ^ f1(D, Kr0_(i), Km0_(i))

(Q(*) is called a "forward quad-round" and QBAR(*) is called a "reverse quad-round".)

（q（*）は「フォワードクアッドラウンド」と呼ばれ、qbar（*）は「逆クアッドラウンド」と呼ばれます。） 

       Let Kr_(i) = {Kr0_(i), Kr1_(i), Kr2_(i), Kr3_(i)} be the set of
       rotation keys for the ith quad-round, where Krj_(i) is a 5-bit
       rotation key for f1, f2, or f3 (as specified above).
        
       Let Km_(i) = {Km0_(i), Km1_(i), Km2_(i), Km3_(i)} be the set of
       masking keys for the ith quad-round, where Kmj_(i) is a 32-bit
       masking key for f1, f2, or f3 (as specified above).
        
       Let KAPPA = (ABCDEFGH) be a 256-bit block where A, B, ..., H are
       each 32 bits in length.
        
       Let "KAPPA <- Wi(KAPPA)" be short-hand notation for the
       following:
           G = G ^ f1(H, Tr0_(i), Tm0_(i))
           F = F ^ f2(G, Tr1_(i), Tm1_(i))
           E = E ^ f3(F, Tr2_(i), Tm2_(i))
           D = D ^ f1(E, Tr3_(i), Tm3_(i))
           C = C ^ f2(D, Tr4_(i), Tm4_(i))
           B = B ^ f3(C, Tr5_(i), Tm5_(i))
           A = A ^ f1(B, Tr6_(i), Tm6_(i))
           H = H ^ f2(A, Tr7_(i), Tm7_(i))

3. Cipher Naming
3. 暗号の命名

In order to avoid confusion when variable keysize operation is used, the name CAST-256 is to be considered synonymous with the name CAST6; this allows a keysize to be appended without ambiguity. Thus, for example, CAST-256 with a 192-bit key is to be referred to as CAST6- 192; where a 256-bit key is explicitly intended, the name CAST6-256 should be used.

可変キーサイズ操作を使用する場合の混乱を避けるために、cast-256という名前は、cast6という名前と同義語と見なされます。これにより、曖昧さなしにキーイジーを追加できます。したがって、たとえば、192ビットキーを持つCAST-256は、CAST6- 192と呼ばれます。256ビットキーが明示的に意図されている場合、名前cast6-256という名前を使用する必要があります。
