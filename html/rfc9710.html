概要

This document provides simple fixes to the IANA "IP Flow Information Export (IPFIX) Entities" registry. Specifically, this document provides updates to fix shortcomings in the description of some Information Elements (IEs), to ensure a consistent structure when citing an existing IANA registry, and to fix broken pointers, orphaned section references, etc. The updates are also meant to bring some consistency among the entries of the registry.

1. Introduction
1. はじめに

When the Operations and Management Area Working Group (OPSAWG) was considering [RFC9565], which obsoletes [RFC7125], the WG realized that some parts of the IANA "IP Flow Information Export (IPFIX) Entities" registry [IANA-IPFIX] were not up to date. This document updates the IANA registry and brings some consistency among the entries of the registry.

オペレーションアンドマネジメントエリアワーキンググループ（OPSAWG）が[RFC9565]を検討していたとき、[RFC7125]が廃止されたとき、WGは、IANAの「IPフロー情報エクスポート（IPFIX）エンティティ」レジストリ[IANA-IPFIX]の一部ではないことに気付きました。最新の。このドキュメントは、IANAレジストリを更新し、レジストリのエントリ間の一貫性をもたらします。

As discussed with IANA during the development of [RFC9487], the "Additional Information" entry in [IANA-IPFIX] should contain a link to an existing registry, when applicable, as opposed to having:

[RFC9487]の開発中にIANAと議論されたように、[IANA-IPFIX]の「追加情報」エントリには、該当する場合は、該当する場合は既存のレジストリへのリンクを含める必要があります。

* A link to an existing registry in the "Description" entry.

* 「説明」エントリの既存のレジストリへのリンク。

* The registry detailed values repeated in the "Description" entry. This practice has the drawback that the description must be updated each time the registry is updated.

* レジストリの詳細値は、「説明」エントリで繰り返されます。このプラクティスには、レジストリが更新されるたびに説明を更新する必要があるという欠点があります。

Therefore, this document lists a set of simple fixes to the IPFIX registry [IANA-IPFIX]. These fixes are classified as follows:

したがって、このドキュメントには、IPFIXレジストリ[IANA-IPFIX]に対する一連の簡単な修正がリストされています。これらの修正は次のように分類されます。

* Updates to fix a shortcoming in the description of an IE (Section 4).

* IE（セクション4）の説明で欠点を修正するための更新。

* Updates to include a pointer to an existing IANA registry (Section 5).

* 既存のIANAレジストリ（セクション5）へのポインターを含めるための更新。

* Updates to ensure a consistent structure when calling an existing IANA registry (Section 6).

* 既存のIANAレジストリを呼び出すときに一貫した構造を確保するための更新（セクション6）。

* Miscellaneous updates to fix broken pointers, orphaned section references, etc. (Section 7).

* 壊れたポインター、孤立したセクション参照などを修正するためのその他の更新（セクション7）。

These updates are also meant to facilitate the automatic extraction of the values maintained in IANA registries (e.g., with a cron job), required by Collectors to be able to support new IPFIX IEs and, more importantly, adequately interpret new values in registries specified by those IPFIX IEs.

これらの更新は、IANAレジストリで維持されている値の自動抽出を促進することも目的としています（例：Cronジョブなど）、コレクターが新しいIPFIX IEをサポートできるようにし、さらに重要なことに、指定されたレジストリの新しい値を適切に解釈することもできます。それらのIPFIX IE。

Note that, as per Section 5 of [RFC7012], [IANA-IPFIX] is the normative reference for the IPFIX IEs that were defined in [RFC5102]. Therefore, the updates in this document do not update any part of [RFC7011].

[RFC7012]のセクション5によると、[IANA-IPFIX]は[RFC5102]で定義されたIPFIX IEの規範的参照であることに注意してください。したがって、このドキュメントの更新は[RFC7011]の一部を更新しません。
