Closing the RTP Payload Format Media Types Registry
RTPペイロードフォーマットメディアタイプレジストリを閉じます
Abstract
概要

The working group process and the authors of RTP payload formats have sometimes failed to ensure that the media types are registered in the IANA "RTP Payload Format Media Types" registry as recommended by RFC 8088. To simplify the process and rely only on the "Media Types" registry, this document closes the RTP payload- specific registry. In addition, it updates the instruction in RFC 8088 to reflect this change.

ワーキンググループプロセスとRTPペイロード形式の著者は、RFC 8088が推奨するIANA「RTPペイロード形式メディアタイプ」レジストリにメディアタイプが登録されていることを保証できない場合があります。プロセスを簡素化し、「メディアタイプ」レジストリにのみ依存するため、このドキュメントはRTPペイロードロードを閉鎖します。さらに、この変更を反映するためにRFC 8088の命令を更新します。

1. Introduction
1. はじめに

Sometimes, authors defining new Real-time Transport Protocol (RTP) payload formats forgot to specify registration of the format's media type in the "RTP Payload Format Media Types" registry [RTP-FORMATS] as recommended by [RFC8088]. In practice, this has no real impact. This registry is not used for any purpose other than to track which media types actually have RTP payload formats, which can be done through other means.

新しいリアルタイムトランスポートプロトコル（RTP）ペイロードフォーマットを定義する著者は、[RFC8088]が推奨する「RTPペイロードフォーマットメディアタイプ」レジストリ[RTPフォーマット]でフォーマットのメディアタイプの登録を指定するのを忘れていました。実際には、これには本当の影響はありません。このレジストリは、どのメディアタイプが実際にRTPペイロードフォーマットを持っているかを追跡する以外の目的には使用されません。これは他の手段で実行できます。

It is required that media types be registered in the "Media Types" registry [MEDIA-TYPES] to identify the format in various signalling usages, avoid collisions, and reference the defining specifications.

メディアタイプを「メディアタイプ」レジストリ[メディアタイプ]に登録して、さまざまなシグナリング使用の形式を識別し、衝突を避け、定義仕様を参照する必要があります。

To resolve this situation, this document:

この状況を解決するために、このドキュメント：

* updates the "RTP Payload Format Media Types" registry to include known RTP payload formats at the time of writing,

* 「RTPペイロードフォーマットメディアタイプ」レジストリを更新して、執筆時点で既知のRTPペイロードフォーマットを含めるようにします。

* closes the "RTP Payload Format Media Types" registry to future registrations and lists this RFC as a reference, and

* 「RTPペイロード形式メディアタイプ」レジストリを将来の登録に閉じ、このRFCを参照としてリストし、

* removes from [RFC8088] the instruction to register RTP payload formats in the "RTP Payload Format Media Types" registry.

* [RFC8088]から「RTPペイロードフォーマットメディアタイプ」レジストリにRTPペイロード形式を登録する命令から削除します。

The origins of the "RTP Payload Format Media Types" registry, as referenced in [RTP-FORMATS], are unclear. The registry cites [RFC4855] as providing the instructions for its maintenance. However, upon reviewing RFC 4855, no text has been found that defines the registry's purpose and operational rules. Further attempts to trace the registry's creation have failed to uncover any references to its establishment. It is likely that the registry was created based on email correspondence or at the request of an Area Director. Consequently, there is no known specification for this registry that requires updating upon its closure.

[rtp-formats]で参照されている「RTPペイロード形式メディアタイプ」レジストリの起源は不明です。レジストリは、[RFC4855]がメンテナンスの指示を提供していると述べています。ただし、RFC 4855をレビューすると、レジストリの目的と運用ルールを定義するテキストは見つかりませんでした。レジストリの作成を追跡しようとするさらなる試みは、その確立への言及を明らかにすることに失敗しました。レジストリは、電子メール通信またはエリアディレクターのリクエストに基づいて作成された可能性があります。その結果、このレジストリの閉鎖時に更新する必要がある既知の仕様はありません。

2. Update to How to Write an RTP Payload Format
2. RTPペイロード形式の書き方の更新

The IANA Considerations section of "How to write an RTP Payload Format" (Section 7.4 of [RFC8088]) mandates that RTP payload formats shall be registered in the "RTP Payload Format Media Types" registry. The following paragraph is updated as shown below, thus removing the need for media types to be registered in the "RTP Payload Format Media Types" registry. Note that this update does not impact the rest of RFC 8088's status as an Informational RFC.

「RTPペイロード形式の書き方」（[RFC8088]のセクション7.4）のIANA考慮事項セクションは、RTPペイロード形式が「RTPペイロード形式メディアタイプ」レジストリに登録されることを義務付けています。次の段落を以下に示すように更新されるため、「RTPペイロード形式のメディアタイプ」レジストリに登録されるメディアタイプの必要性が削除されます。この更新は、情報RFCとしてのRFC 8088の残りのステータスに影響を与えないことに注意してください。

OLD:

OLD:

Since all RTP payload formats contain a media type specification, they also need an IANA Considerations section. The media type name must be registered, and this is done by requesting that IANA register that media name. When that registration request is written, it shall also be requested that the media type is included under the "RTP Payload Format media types" sub-registry of the RTP registry (http://www.iana.org/assignments/rtp-parameters).

すべてのRTPペイロード形式にはメディアタイプの仕様が含まれているため、IANAの考慮事項セクションも必要です。メディアタイプ名を登録する必要があり、これはIANAにそのメディア名を登録するように要求することによって行われます。その登録リクエストが書かれている場合、メディアタイプは「RTPペイロードフォーマットメディアタイプ」RTPレジストリ（http://www.iana.org/assignments/rtp-parameters）のサブレジストリに含まれることも要求されます。

NEW:

NEW:

Since all RTP payload formats contain a media type specification, they also need an IANA Considerations section. The media type name must be registered, and this is done by requesting that IANA register that media name in the "Media Types" registry (https://www.iana.org/assignments/media-types/).

すべてのRTPペイロード形式にはメディアタイプの仕様が含まれているため、IANAの考慮事項セクションも必要です。メディアタイプ名を登録する必要があります。これは、IANAに「メディアタイプ」レジストリ（https://www.iana.org/assignments/media-types/）にそのメディア名を登録するよう要求することによって行われます。
